Atom

发表于 2015-06-27   |  

在Mac上陆续尝试过一些编辑器，从最早的TextMate，Sublime Text，到Light Table，当然还有比较重量级的如Xcode、Eclipse、Intellij IDEA。这些编辑器各有优势，如今认识了Atom，被它Geek风格吸引，一些特性能使日常工作更加高效。

支持多种代码的Autocomplete

因为日常工作中经常要多种语言间切换，这个功能在一个轻量级的编辑器中支持尤为重要，基本需求就是能根据
文件名后缀，自动识别语言，并启动AutoComplete功能。这个在Atom中支持的不错

  • c/c++ 的自动提示

  • python 自动提示

支持Ctags

这个是c/c++必不可少的插件了，atom-ctags可以支持

支持多种语言的快速编译执行

经常在几种语言间切换，shell, c/c++，python，java, js，最需要的就是能一个快捷键下去快速编译执行代码片段。这点
通过Atom的Script插件可以较好支持

源码的全屏视图模式

这个在Sublime Text中第一次尝试，没想到Atom默认也支持了，查看大文件时，还是蛮有用的.

支持markdown书写及即时预览

MarkDown现在是主要使用的文档语言, 之前都是在Mou、Ulyssess来书写，现在Atom中直接支持，很是方便.

Ctrl + Shift + M 即可呼唤中即时预览功能

支持Hexo

Hexo是一个静态博客系统，可以直接支持在Github上搭建静态博客，在Atom中可以通过atom-hexo插件方便使用hexo.

支持Shell

编辑器直接集成终端，边写shell，边终端查看，会是一件很方便的事情，Termrk、term2都可以支持

支持vim编辑方式

vim编辑模式已经是一种习惯了，以往尝试任何编辑器都会看看支持支持vim编辑，Atom在这点上做的不错。vim-mode插件可以支持.

swift学习(二)

发表于 2015-06-14   |  

Swift中有三种方式可以表示“任意”: Any、AnyObject以及AnyClass. 在Swift中内部实现中，是采用typealias来定义AnyClass的.

typealias AnyClass = AnyObject.Type

我们可以通过如下代码来获取类的类型:

class A {

}

let a: A.Type = A.self

.self 可以跟在类名后面得到类型本身，也可以跟在实例后面，得到实例本身。像上面的写法就等价于下面这种用法:

class A {

}

let a: AnyClass = A.self

比如复杂一点的用法是如下：定义一个类方法:

class A {
class func method() {

}
}

let a: A.Type = A.self
a.method()

let b: AnyClass = A.self
(b as! A.Type).method()

这样的使用方式没有A.method()来得方便，但是AnyClass可以编程灵活不少，另外，在Cocoa中的一些方法也是需要我们以AnyClass的方式调用，比如：

collection.registerClass(NTWaterfallViewCell.self, forCellWithReuseIdentifier: waterfallViewCellIdentify)

Type可以用于: class, struct, enum以及protocol上面.

重温swift

发表于 2015-06-14   |  

Apple开发一直是个梦想，又是好几个月没有碰了，今天重新拿起一个具体工程项目PinterestSwift进行学习。 现在Swift已经到了1.2， 先从语法开始。扫了下整个工程代码，记录如下补充学习：

（1）在Swift1.2 中，向上转化(upcast)、向下转化(downcast)时用到as关键词，需要在后面加上问号(?)，或者感叹号(!)，这样编译时就能得到提示。

1.2以前没有这个功能，所以转化错误时，只能在运行期间发现。

（2）1.2中，NSString是无法自动转化为String的，需要加上as String

（3）swift中的Protocol所有的方法都必须实现，这点和Cocoa中的有不同，那么如何在wsift中定义像Object-c中一样的可选方法呢？ 答案就是加入@objc 关键词， 比如： 

@objc protocol OptionalProtocol { optional func optionalMethod() }

需要注意的是，这里的optionalMethod()方法是可选的，因为前面有optional关键词，如果没有这个词，那么这个方法还是必须被实现。且这个OptionalProtocol协议必须被Class实现，不能被struct或者Enum来实现。

（4）layoutSubviews()方法何时被调用？

  • init初始化不会触发layoutSubviews
  • addSubview会触发layoutSubviews
  • 设置view的Frame会触发layoutSubviews，当然前提是frame的值设置前后发生了变化
  • 滚动一个UIScrollView会触发layoutSubviews
  • 旋转Screen会触发父UIView上的layoutSubviews事件
  • 改变一个UIView大小的时候也会触发父UIView上的layoutSubviews事件

（5）Swift是自动管理内存的，当我们创建一个对象是，swift会自动增加引用计数，当我们释放一个对象时，swift会自动减小引用计算，所有自动管理内存都由会一个问题，循环引用（retain Cycle）。比如如下： 

class A { 
    let b: B 
    init() { 
        b = B() 
        b.a = self 
    } 

    deinit { 
        println("A deinit") 
    } 
} 

class B { 
    var a: A? = nil 
    deinit { 
        println("B deinit") 
    } 
}

上述就是一个典型的循环引用，A初始化时会创建B，同时让B引用A。A和B相互引用。这样当第三方调用时，引用计数永远不会减小。造成循环引用。如何来避免呢？ 

class B { 
    weak var a: A? = nil 
    deinit { 
        println("B deinit") 
    } 
}

在B中弱引用A，这样当有删除A时，B会优先告诉内存管理，我不再引用A了。

一般的用法，未delegate属性设置一个weak属性

（6）willSet{}是在属性被存储之前调用，didSet{}是在属性被存储之后调用。

（7）在类中，可以在init初始化方法中为一个常量赋值，这是重要的一个特性，比如： 

class ClassA { 
    let numA: Int 
    init(num: Int) { 
        numA = num 
    } 
} 

class ClassB: ClassA { 
    let numB: Int 

    override init(num: Int) { 
        numB = num + 1 
        super.init(num: num) 
    } 
}

如果想让子类务必实现一个init方法，可以强制required关键词。 required关键词的作用如下：

  • 只能修饰类的初始化方法，init
  • 父类中有required修饰的init()时，子类也有多态的init()时， 必须重载父类init，否则编译错误。
  • 如果子类中没有init，那么可以不需要重载父类的init

迁移至hexo

发表于 2015-05-31   |  

几年前使用octopress，一款用ruby写的静态博客系统，可以方便的部署在github上，第一次碰到时被它简单所吸引。但每次安装配置octopress，真是要命，在mac上的高版本Ruby总是有问题，需要依赖1.9.3版本的Ruby，安装很不顺利。

今天看到Hexo，基于node.js写的静态博客系统，使用起来比octopress方便不少，果断迁移过去。

具体的安装部署方法：

  • 首先需要安装Node.js。 点击这里下载
  • 下载安装hexo

npm install -g hexo-cli
npm install hexo —save
npm install hexo-generator-index —save
npm install hexo-generator-archive —save
npm install hexo-generator-category —save
npm install hexo-generator-tag —save
npm install hexo-server —save
npm install hexo-deployer-git —save
npm install hexo-deployer-heroku —save
npm install hexo-deployer-rsync —save
npm install hexo-deployer-openshift —save
npm install hexo-renderer-marked —save
npm install hexo-renderer-stylus —save
npm install hexo-generator-feed —save
npm install hexo-generator-sitemap —save

  • 本地建立一个空目录，进入目录，执行

hexo init

  • 新建文章

hexo n 文章标题

  • 产生静态页面

hexo g

  • 启动server，预览文章

hexo s

  • 确认OK后，可以push到github上

hexo deploy

执行deploy前，需要对_config.yml文件中进行如下正确设置：

deploy:
type: git
repository: https://github.com/vbarter/vbarter.github.io.git
branch: master

最后可以设置自己域名绑定github page上，需要注意的是一定确保分支下有CNAME设定好自己的域名

香港一卡通

发表于 2013-08-31   |   分类于 其它   |  

如何走向世界，先从招商银行香港一卡通开始把，有了它，你可以自由的将人民币换成美元，然后转账到香港一卡通，最后汇款到世界任何一家银行。而这一切都只要在网络上操作即可完成。从办理到拿到卡片大概需要两周时间。

（插一句，没有香港一卡通钱，为了汇点钱到国外银行，我估计跑了不下十次银行，成功一次，其它全部失败，各种限制不给汇 。）

这里说明下如何才能办到香港一卡通，购汇和汇款。

材料准备

这个网上一查就可以知道，需要准备好如下：

（1）招商银行的金卡，存款5万以上即可申请。顺便办一个USB KEY

（2）房本或其它地址证明材料（这个我没试过），建议最好带房本原件。

（3）港澳通信证或者护照。

（4）身份证。

办理过程

在北京分行北京分行西三环支行北京市海淀区阜成路67号，银都大厦一层，航天桥向西200米路北（010-68718147/68718141）办的，第一次证件还不全（忘记了房本），还跑了2次。按照步骤填写表格。等一会办完后，银行就会给你一个回执单，上面有香港一卡通的号码和验证码，大概一天后即可以收到香港那边短信通知，已经开通账户，然后需要做如下几件事情：

（1）下载香港招商银行专业版，http://hk.cmbchina.com这里下载

（2）使用你金卡的USB KEY登陆香港专业版，第一次登陆会提示你绑定，输入回执单中的验证码即可绑定。

（3）因为要从内地转账到香港一卡通需要开通“证劵账户”，而开通“证劵账户”之前，你需要完善更新的你用户信息，点击“其他”-> “更改客户资料”，更新后等吧，运气好的话，一个工作日后生效。

（4）一天后，你会收到短信提示你的个人资料已经审核通过，你可以申请，点击“证劵服务” -> “投资主页” 进去按照提示申请，运气好的话，一个工作日后生效。

（5）当证劵账户也开通后，基本万事具备了，现在就差将人民币转化为美元，转到香港一卡通了。

内地转账到香港一卡通

第一次可能麻烦点，估计第二次后，几分钟就可以完成全部操作了。

（1）可以登陆网上银行，“外汇管理” -> “购汇”

（2）完成购汇以后，你内地卡就用了美元，可以转到香港，用招商银行的专业版。 “转账汇款” -> “境外汇款” -> “汇香港一卡通”。关键信息如下，其他信息自己根据提示写。

收款姓名：填写拼音，姓和名之间需要空格分开。
收款账户：香港一卡通的账号。
收款方地址：HONGKONG
币种：美元  现汇
交易编码：随便选，可以选择培训费用。
交易附言：随便写。
汇款附言：随便写。

（3）点击确定后，就可以把美元转到香港一卡通了。

香港一卡通汇款到国外银行

直接用香港一卡通的专业版可以完成。“转账汇款” -> “国外汇款”，按照提示填写即可，咨询了下客户，我汇款到sogotrade不需要填写swift码和银行地址，但如果在银行柜台办理，这两项必填。

sentry c/c++ client

发表于 2013-07-24   |   分类于 c++   |  

sentry是一个python实现的开源信息收集服务，为它实现了一个c/c++客户端sentry-cpp，这样就可以在c++工程中把信息发送给sentry.

sentry-cpp is a c/c++ client for Sentry

include <Sentry.h>

int main(int argc, char\*\* argv) 
{
//Instantiate a new client with a compatible DSN
SENTRY.init("http://public:secret@example.com/1");
//only title
SENTRY.error("title");
//title and message
SENTRY.info("title","message");
//send extra data
Json::Value extra;
SENTRY.info("title","message",&extra);
}

Dependence

before you use sentry-cpp, you need install the following libraries.

  1. Jsoncpp

  2. libcurl

Install

next, you need edit the Makefile, and change the path for json-cpp and libcurl, now you can

make

it will generate static library(.a) and share library(.so), you can use sentry-cpp in three ways:

  1. use source code, only Sentry.cpp and Sentry.h

  2. use shared object

  3. use static object

Enjoy!
